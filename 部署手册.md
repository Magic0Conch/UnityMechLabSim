# 电火花加工虚拟仿真实验系统 — 部署手册

本文档面向**系统管理员与运维人员**，说明如何将后端部署到**学校公共服务器**（**Linux 或 Windows Server** 均可），并配置 Unity 客户端连接。教师与学生操作请参阅《用户使用手册.md》。

---

## 一、架构概览

```
                    ┌─────────────────────────────────┐
  校园网 / 公网      │  公共服务器（Linux / Windows）    │
                    │         ┌────────────┐          │
  Unity 客户端 ────►│         │ Uvicorn      │          │
  (学生/教师 PC)    │         │ FastAPI :8899│          │
                    │         └──────┬───────┘          │
                    │                │                  │
                    │         ┌──────▼───────┐          │
                    │         │ SQLite + 文件 │          │
                    │         │ server/data/ │          │
                    │         └──────────────┘          │
                    └─────────────────────────────────┘
```

| 组件 | 说明 |
|------|------|
| **FastAPI 后端** | 目录 `server/`，兼容 Unity `requesttype` 协议 |
| **入口地址** | `http://<服务器IP>:8899/api/values` |
| **静态 PDF** | `http://<服务器IP>:8899/files/{名称}`（如 m2、m3） |
| **健康检查** | `http://<服务器IP>:8899/health` |
| **数据目录** | `server/data/`（数据库 + 上传文件，须定期备份） |

## 二、服务器要求

### 2.1 硬件与系统

| 项目 | 建议 |
|------|------|
| 操作系统 | **Ubuntu 22.04 LTS**、CentOS 7+ / Rocky Linux 8+，或 **Windows Server 2016 / 2019 / 2022** |
| CPU / 内存 | 2 核、2 GB 起（班级并发上传建议 4 GB+） |
| 磁盘 | 50 GB+（视学生作业量调整） |
| Python | **3.10 及以上** |
| 网络 | 固定 **公网 IP**（或校内 IP）；云控制台安全组与系统防火墙放行 **8899** |

### 2.2 客户端连接地址

本系统按 **IP + 端口** 直连部署，无需域名与 HTTPS：

- 示例：`http://49.233.160.65:8899/api/values`（将 IP 换为你的服务器）
- Unity 登录页、健康检查、PDF 资源均使用同一主机与端口，**无需**单独配置

## 三、Linux 服务器部署

以下以 Ubuntu 22.04、部署用户 `mechlab`、代码目录 `/opt/mechlab` 为例。请按实际环境替换。

> **使用 Windows Server？** 请直接阅读 [第四节 Windows Server 部署](#四windows-server-部署)。

### 3.1 上传代码

将工程中的 `server/` 目录上传到服务器，例如：

```bash
# 本机示例（需已配置 SSH）
scp -r server/ mechlab@your-server:/opt/mechlab/server/
```

服务器上目录结构：

```
/opt/mechlab/server/
├── app/
├── requirements.txt
└── data/          # 首次启动后自动生成，亦可预先创建
```

### 3.2 安装 Python 环境

```bash
sudo apt update
sudo apt install -y python3 python3-venv python3-pip

cd /opt/mechlab/server
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

### 3.3 首次启动与自检

```bash
cd /opt/mechlab/server
source .venv/bin/activate
uvicorn app.main:app --host 0.0.0.0 --port 8899
```

另开终端测试：

```bash
curl http://127.0.0.1:8899/health
# 期望: {"status":"ok"}
```

本机浏览器也可试：`http://<服务器IP>:8899/health`。

确认无误后 `Ctrl+C` 停止，继续配置 systemd。

### 3.4 配置 systemd 常驻服务

创建 `/etc/systemd/system/mechlab.service`：

```ini
[Unit]
Description=MechLab FastAPI Backend
After=network.target

[Service]
Type=simple
User=mechlab
Group=mechlab
WorkingDirectory=/opt/mechlab/server
Environment="PATH=/opt/mechlab/server/.venv/bin"
ExecStart=/opt/mechlab/server/.venv/bin/uvicorn app.main:app --host 0.0.0.0 --port 8899 --workers 2
Restart=always
RestartSec=5

[Install]
WantedBy=multi-user.target
```

```bash
sudo chown -R mechlab:mechlab /opt/mechlab
sudo systemctl daemon-reload
sudo systemctl enable mechlab
sudo systemctl start mechlab
sudo systemctl status mechlab
```

> 生产环境**不要**使用 `--reload`。`--host 0.0.0.0` 表示接受来自校园网/公网的连接。并发较高时可适当增加 `--workers`（通常为 CPU 核数）。

### 3.5 防火墙与安全组

**系统防火墙（Ubuntu ufw）：**

```bash
sudo ufw allow OpenSSH
sudo ufw allow 8899/tcp
sudo ufw enable
```

**云控制台安全组**（腾讯云等）：入站规则放行 **TCP 8899**，来源可按需设为 `0.0.0.0/0`（全校访问）或学校网段。

部署完成后，在任意电脑浏览器访问 `http://<服务器IP>:8899/health` 应返回 `{"status":"ok"}`。

## 四、Windows Server 部署

后端为纯 Python 应用，**完全支持在 Windows Server 上运行**，与 Linux 使用同一套 `server/` 代码与 `server/data/` 数据目录。学校机房若已有 Windows Server，无需为了本系统单独购置 Linux 主机。

### 4.1 适用版本与环境

| 项目 | 说明 |
|------|------|
| 系统 | Windows Server **2016 / 2019 / 2022**（桌面版 Windows 10/11 仅适合调试，不建议作生产机） |
| Python | **3.10+**（从 [python.org](https://www.python.org/downloads/) 安装，勾选 *Add Python to PATH*） |
| 部署目录示例 | `C:\mechlab\server\` |
| 默认端口 | **8899**（避开 Windows 保留端口 5040 等） |

### 4.2 安装与首次验证

将工程中的 `server` 文件夹复制到服务器，在 **PowerShell（管理员）** 中执行：

```powershell
cd C:\mechlab\server
python -m venv .venv
.\.venv\Scripts\Activate.ps1
pip install -r requirements.txt

# 前台试跑（确认无报错后 Ctrl+C 停止）
.\.venv\Scripts\uvicorn.exe app.main:app --host 127.0.0.1 --port 8899
```

另开 PowerShell 测试：

```powershell
Invoke-WebRequest http://127.0.0.1:8899/health
# 应返回 StatusCode 200，内容为 {"status":"ok"}
```

### 4.3 注册为 Windows 服务（推荐，开机自启）

使用 [NSSM](https://nssm.cc/)（Non-Sucking Service Manager）将 Uvicorn 注册为系统服务：

```powershell
# 假设 nssm.exe 已放入 C:\tools\nssm\nssm.exe
cd C:\tools\nssm

.\nssm.exe install MechLab "C:\mechlab\server\.venv\Scripts\uvicorn.exe"
.\nssm.exe set MechLab AppDirectory "C:\mechlab\server"
.\nssm.exe set MechLab AppParameters "app.main:app --host 0.0.0.0 --port 8899 --workers 2"
.\nssm.exe set MechLab DisplayName "MechLab 实验后端"
.\nssm.exe set MechLab Start SERVICE_AUTO_START

.\nssm.exe start MechLab
```

查看状态：

```powershell
Get-Service MechLab
```

卸载服务：`nssm remove MechLab confirm`

**备选**：不用 NSSM 时，可用「任务计划程序」在系统启动时运行 `start_mechlab.bat`：

```bat
@echo off
cd /d C:\mechlab\server
.\.venv\Scripts\uvicorn.exe app.main:app --host 0.0.0.0 --port 8899 --workers 2
```

### 4.4 防火墙与对外访问

服务监听 `0.0.0.0:8899`，由 Windows 防火墙对校园网/公网开放端口：

```powershell
New-NetFirewallRule -DisplayName "MechLab API 8899" `
  -Direction Inbound -Protocol TCP -LocalPort 8899 -Action Allow
```

Unity 客户端地址：`http://<服务器IP>:8899/api/values`

### 4.5 数据目录与备份

与 Linux 相同，所有持久化数据在：

```
C:\mechlab\server\data\
├── mechlab.db
└── storage\
    ├── common\
    └── uploads\
```

定期备份（任务计划程序 + PowerShell 示例）：

```powershell
$src = "C:\mechlab\server\data"
$dst = "D:\backup\mechlab-data-$(Get-Date -Format yyyyMMdd).zip"
Compress-Archive -Path $src -DestinationPath $dst -Force
```

### 4.6 预置资源与题库

与 Linux 相同：教师端 U 菜单上传，或将 `教师示范` 目录复制到服务器后执行：

```powershell
cd C:\mechlab\教师示范
C:\mechlab\server\.venv\Scripts\python.exe sync_to_server.py
```

### 4.7 Windows 与 Linux 选型建议

| 场景 | 建议 |
|------|------|
| 学校已有 Windows Server | **Windows Server + 8899 直连**（第四节） |
| 云主机、运维习惯 Linux | **Ubuntu + 8899 直连**（第三节） |
| 仅本机或单机调试 | Windows 10/11 直接 `uvicorn`（见第十二节） |

两种系统下 Unity 客户端**仅修改登录页服务器地址**，协议与接口完全一致。

## 五、初始化数据

### 5.1 自动初始化

首次启动时，`init_db()` 会创建 SQLite 数据库并写入演示账号：

| 用户名 | 密码 | 角色 |
|--------|------|------|
| teacher | teacher123 | 教师 |
| student1 | 123456 | 学生 |
| student2 | 123456 | 学生 |

并在 `server/data/storage/common/` 生成占位公共文件。

学生首次用不少于 6 位的学号登录时，若库中无该用户会**自动建号**（无需事先导入）。

### 5.2 上线前必做：修改教师密码

演示账号**不得**用于生产环境。请通过教师端导入真实学生、修改教师密码，或直接在数据库中更新：

```bash
sqlite3 /opt/mechlab/server/data/mechlab.db \
  "UPDATE users SET password='新强密码' WHERE username='teacher';"
```

### 5.3 预置教学资源

将 PDF、zip、题库等放入公共存储并登记数据库。方式一：教师登录客户端，在 Tscene 的 **U** 菜单上传（推荐，与日常一致）。

方式二：在服务器上同步示范题库（需将 `教师示范/` 一并上传）：

```bash
cd /opt/mechlab/教师示范   # 若已上传该目录
/opt/mechlab/server/.venv/bin/python sync_to_server.py
/opt/mechlab/server/.venv/bin/python verify_sync.py
```

手动放置公共文件时，文件放在 `server/data/storage/common/`，并在 `common_files` 表中登记 `name` 与 `storage_path`。

### 5.4 公共文件命名约定

| 逻辑名称 | 用途 |
|----------|------|
| 实验目的内容 | 学生任务 0 下载 zip |
| 实验安全考试 | 安全操作考试题库 |
| 职工素养测试 | 素养考试题库 |
| m2、m3 | PDF（设备仪器、实验指导书） |
| videos | 视频链接文本 |
| 实验指导书、实验报告模板 | 教师可选公共资源 |

## 六、客户端配置与分发

### 6.1 用户自行填写地址

发布客户端后，用户在登录页填写（将 IP 换成实际服务器）：

```
http://49.233.160.65:8899/api/values
```

地址会保存在本机，换机需重新填写。

### 6.2 打包时写入默认地址（推荐）

在发布 Unity 构建前，修改 `Assets/Script/newScript/LoginUI.cs`：

```csharp
public string InitConnectUri = "http://49.233.160.65:8899/api/values";
```

重新构建 Windows 发布版，分发给机房或学生。用户仍可在登录页修改地址。

## 七、数据目录与备份

### 7.1 目录说明

| 路径 | 内容 |
|------|------|
| `server/data/mechlab.db` | 用户、成绩、提交记录元数据 |
| `server/data/storage/common/` | 教师上传的公共文件 |
| `server/data/storage/uploads/<用户名>/` | 学生作业文件 |

### 7.2 备份策略

建议每日备份 `server/data/` 整个目录：

```bash
#!/bin/bash
# /opt/mechlab/backup.sh
tar -czf /backup/mechlab-data-$(date +%Y%m%d).tar.gz -C /opt/mechlab/server data
```

可加入 crontab：

```bash
0 2 * * * /opt/mechlab/backup.sh
```

恢复时停止服务 → 解压覆盖 `data/` → 重启 `mechlab` 服务。

## 八、运维与监控

### 8.1 常用命令

```bash
# 服务状态
sudo systemctl status mechlab

# 查看日志
sudo journalctl -u mechlab -f

# 重载配置后重启
sudo systemctl restart mechlab
```

Windows Server：

```powershell
Get-Service MechLab
Restart-Service MechLab
# NSSM 日志默认在 C:\tools\nssm\service\MechLab\output.log
```

### 8.2 健康检查

```bash
curl -s http://49.233.160.65:8899/health
```

可配置监控系统定期请求 `/health`。

### 8.3 API 抽测

```bash
# 登录
curl -X POST "http://49.233.160.65:8899/api/values" \
  -F "requesttype=LOGIN" \
  -F "username=student1" \
  -F "password=123456"

# 学生进度
curl -X POST "http://49.233.160.65:8899/api/values" \
  -F "requesttype=GEIINFO" \
  -F "username=student1"
```

### 8.4 升级后端

```bash
cd /opt/mechlab/server
source .venv/bin/activate
# 更新 app/ 代码后
pip install -r requirements.txt
sudo systemctl restart mechlab
```

升级前请先备份 `server/data/`。

## 九、安全建议

| 项 | 说明 |
|----|------|
| 传输 | 当前为 HTTP 明文；仅限教学场景，敏感环境请评估风险 |
| 教师密码 | 上线后立即修改 `teacher` 默认密码 |
| 端口 | 对外开放 **8899**；云安全组与 ufw 同步配置 |
| 备份 | 定期备份 `data/`，并做恢复演练 |
| 磁盘 | 监控 `uploads/` 体积，防止占满磁盘 |
| 访问控制 | 云安全组可限制来源网段，仅允许校内 IP 访问 |

当前后端为教学场景轻量实现：密码明文存储、无复杂鉴权。若面向大范围公网，请评估学校信息安全规定并考虑加固（统一认证、密码哈希、审计日志等）。

## 十、故障排查

| 现象 | 排查 |
|------|------|
| `/health` 无响应 | Linux：`systemctl status mechlab`；Windows：`Get-Service MechLab`；检查 8899 是否监听 |
| 外网无法访问 | `systemctl status mechlab`；云安全组与 ufw 是否放行 **8899** |
| 上传失败 / 超时 | 后端单文件上限约 500MB；检查网络与磁盘空间 |
| PDF 打不开 | `common/` 下是否存在 m2、m3 文件；`common_files` 表是否登记 |
| 登录 FAILED | 数据库是否初始化；用户名密码是否正确 |
| Windows 本机 5040 端口占用 | 开发环境请用 **8899**，与生产配置保持一致 |
